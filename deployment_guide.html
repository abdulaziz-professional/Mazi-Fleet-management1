Deployment guide

Mazi Fleet Dashboard on GitHub Pages

Use this guide to publish index.html, connect a public Google Sheet CSV feed, and train staff to refresh the dashboard without touching code.

What you have

The dashboard is a static HTML file named index.html. It uses Chart.js and Papa Parse from public CDNs, stores the Google Sheet URL in browser localStorage, and auto-refreshes the CSV every 60 seconds while the page is open.

1. Prepare the Google Sheet

Create a Google Sheet with the exact column headers in row 1. The dashboard is forgiving about minor spacing and capitalization, but the safest setup is to use these names exactly:

SL, NoPlate, Model, Make, Year, Colour, Customer, Nationality, Contact, Start Date, End Date, Actual Rent, Security Deposit, Collected, Payment Status, Status, Type, Standard Rent

2. Make the sheet accessible as CSV

Important: Publishing or public sharing can expose spreadsheet data to anyone with access to the link. Remove private or sensitive information before publishing.

Recommended method: Publish to web

Open the sheet on a computer.

Use Google Sheets in a desktop browser.

Go to File > Share > Publish to web.

Select the tab that contains the fleet data. Publish the full spreadsheet only when every tab is safe to expose.

Choose the CSV or comma-separated format if offered.

If Google gives you a published link, copy it. The dashboard can accept published links such as /pub?output=csv or /pubhtml.

Paste that link into the dashboard.

Click Save & Load. The link is saved in that browser so staff only paste it once.

Alternative method: Share link

Set the sheet to Anyone with the link can view, copy the normal Sheets URL, and paste it into the dashboard. The dashboard will try to convert it into:

https://docs.google.com/spreadsheets/d/SHEET_ID/export?format=csv&gid=TAB_GID

This method is convenient, but the published CSV method is usually easier to troubleshoot because the CSV URL can be tested directly in an incognito browser tab.

3. Deploy index.html to GitHub Pages

Create or open a GitHub repository.

For a project site, any repository name is fine. For a user site, the repository name is usually username.github.io.

Upload the dashboard file.

Place index.html at the root of the repository, or at the root of the folder you will publish from. Keep the filename exactly index.html.

Open Settings > Pages.

Under Build and deployment, choose Deploy from a branch.

Select the branch and folder.

Choose main and /(root) when index.html is in the repository root. Save the Pages settings.

Open the published site.

GitHub will show a Pages URL after deployment. Publishing can take several minutes after changes are pushed.

Done: Open the GitHub Pages URL, paste the Google Sheet URL, click Save & Load, and the dashboard will refresh every 60 seconds while open.

4. Staff operating steps

Open the GitHub Pages dashboard URL.

Use Chrome, Edge, Safari, or Firefox.

Paste the Google Sheet URL once.

Click Save & Load. The URL is saved only in that browser profile.

Update rows in Google Sheets.

The page checks for fresh CSV data every 60 seconds. Staff can also click Refresh Now.

Use slicers for analysis.

Set date range, model, nationality, colour, status, payment, and year, then click Apply. Click Reset to return to all records.

5. Troubleshooting

ProblemLikely causeFix
Dashboard says the link returned HTMLThe sheet is private, the URL is not a CSV endpoint, or Google returned a sign-in page.Publish to web, copy the CSV/published link, and test it in an incognito tab. The tab should show plain CSV text, not a web page.

Charts show wrong monthsDate format is inconsistent.Use DD/MM/YYYY or YYYY-MM-DD consistently.
Wrong tab is loadedThe copied URL has the wrong gid.Open the correct sheet tab, copy the URL again, and save it in the dashboard.
Staff need to change the sourceThe saved link is stored in localStorage on that browser.Click Clear Saved Link, paste the new Sheet URL, then click Save & Load.
