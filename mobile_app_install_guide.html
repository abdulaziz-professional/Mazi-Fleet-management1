Mobile App Setup Guide

Install the Mazi Fleet Dashboard like an Android/iOS app using Progressive Web App files hosted on GitHub Pages.

Files to upload to GitHub

Upload all of these to the root of the same GitHub repository where your dashboard is hosted.

index.html
manifest.webmanifest
sw.js
icons/icon-192.png
icons/icon-512.png
icons/apple-touch-icon.png

Update GitHub Pages

  1. Open your GitHub dashboard repository.
  2. Click Add file, then Upload files.
  3. Drag and drop the complete package files. Keep the icons folder as a folder.
  4. If GitHub asks to replace old index.html, allow the replacement.
  5. Scroll down and click Commit changes.
  6. Open your GitHub Pages dashboard URL after the commit finishes.

Install on Android

  1. Open the GitHub Pages dashboard link in Chrome on the Android phone.
  2. Wait until the page finishes loading.
  3. Tap the dashboard's Install App button. If the install prompt does not appear, open Chrome menu.
  4. Tap Install app or Add to Home screen.
  5. Confirm the name Mazi Fleet and tap Install.
  6. Open the app from the phone home screen or app drawer.

Install on iPhone or iPad

  1. Open the GitHub Pages dashboard link in Safari.
  2. Tap the Share button.
  3. Scroll and tap Add to Home Screen.
  4. Turn on Open as Web App if the option appears.
  5. Tap Add.
  6. Open Mazi Fleet from the iPhone or iPad home screen.

Connect both Google Sheets

  1. Open the installed app or normal browser dashboard.
  2. Paste the Dubai Google Sheet URL in Dubai number plate sheet URL.
  3. Paste the Sharjah Google Sheet URL in Sharjah number plate sheet URL.
  4. Click Save Both & Load.
  5. Check the table Source column to confirm Dubai and Sharjah rows are combined.

The dashboard saves the sheet links separately in the device browser storage, so staff paste the URLs once per device/browser.

Troubleshooting

If the app does not install, check that your dashboard is opened from the live GitHub Pages HTTPS URL, not from a local file on your computer.

If the dashboard opens but data does not load, make sure both Google Sheets are shared as public viewer links or published to the web as CSV-compatible sheets.
